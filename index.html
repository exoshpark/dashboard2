## 허브 연동 방식
대시보드는 허브에서 열리며 URL hash로 API 키와 언어값을 전달받습니다.

```javascript
#ak=ACCESS_KEY&sk=SECRET_KEY&lang=ko
```

아래 코드를 </body> 바로 위 <script> 블록 끝에 포함해주세요.

```javascript
(async function initFromHub() {
  var _ak = '', _sk = '', _lang = 'ko';
  try {
    var _p = new URLSearchParams(location.hash.slice(1));
    _ak = _p.get('ak') || '';
    _sk = _p.get('sk') || '';
    _lang = _p.get('lang') || 'ko';
  } catch(e) {}

  if (!_ak || !_sk) {
    try {
      _ak = localStorage.getItem('exosp_access') || '';
      _sk = localStorage.getItem('exosp_secret') || '';
    } catch(e) {}
  } else {
    try {
      localStorage.setItem('exosp_access', _ak);
      localStorage.setItem('exosp_secret', _sk);
    } catch(e) {}
  }

  window.EXOSP_LANG = _lang;
  if (typeof applyTranslations === 'function') applyTranslations();

  if (!_ak || !_sk) return;
  await connectToApi(_ak, _sk);
})();
```

## 키 저장
- 허브에서 `ak`, `sk`가 전달되면 브라우저 `localStorage`에 저장해주세요.
- hash에 키가 없으면 `localStorage`의 `exosp_access`, `exosp_secret` 값을 사용해주세요.
- 키가 없을 때는 API 호출을 하지 않고 빈 상태 또는 데모 안내 상태를 보여주세요.

## 다국어 적용
- `window.EXOSP_LANG` 기준으로 `ko`, `en`, `ja`를 지원해주세요.
- 번역 객체를 만들고 `applyTranslations()` 함수에서 화면 텍스트를 갱신해주세요.
- 정적 텍스트뿐 아니라 차트 라벨, 테이블 헤더, 상태 배지, 빈 상태 문구도 다국어 처리해주세요.
- 데이터 렌더링 함수가 있다면 언어 변경 후 다시 렌더링되도록 해주세요.
- 허브에서 언어가 변경될 수 있으므로 아래 이벤트를 지원해주세요.

```javascript
window.addEventListener('message', function(e){
  if(e.data && e.data.type === 'EXOSP_LANG'){
```
